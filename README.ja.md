# 研墨 / Yanmo

[![Tests](https://img.shields.io/badge/tests-49%20passed-green)](https://github.com/sixtdreanight/Yanmo/actions)
[![Python](https://img.shields.io/badge/python-3.11+-blue)](https://www.python.org/)
[![License](https://img.shields.io/badge/license-MIT-green)](LICENSE)
[![Platform](https://img.shields.io/badge/platform-Windows%20%7C%20Linux%20%7C%20macOS-lightgrey)]()

**あなたのコンピュータ上で動作し、データが外部に出ない研究アシスタント。**

汎用AIチャットとは違います。研墨は研究を行う人のために設計されています：指導教員の曖昧な指示を分解し、複数情報源から最新論文を追跡し、数式の導出を相互検証し、プロジェクト計画を客観的に評価し、論文執筆を支援します。5つのツールはそれぞれ独自のインターフェースを持ち、一つのチャットボックスに詰め込んだ妥協はしていません。

核心は3つ：**データはローカルに留まる、数式の誤りは許さない、独自プラグインをサポート。** AI重複除去、AI発想支援、AI臭の除去機能も付属しています。

---

## インストールと実行

Python 3.11+、Node.js 18+ および Ollama が必要です。

```bash
pip install -e ".[dev]"
python -m backend.main
```

バックエンドは `127.0.0.1:8000` で起動します。次にフロントエンドを起動します：

```bash
cd frontend
npm install
npm run dev
```

ブラウザで `http://localhost:5173` を開きます。

### デスクトップアプリケーション

> **注意 (2026-05):** メインでメンテナンスされているフロントエンドは React Web アプリケーション (`frontend/`) です。
> Tauri デスクトップシェル (`frontend/src-tauri/`) および Qt フロントエンド (`frontend_qt/`) は
> **非推奨**となり、将来のバージョンで削除される予定です。3つのフロントエンドを維持することは小規模チームには持続不可能です。
> Tauri または Qt フロントエンドに依存している場合は、Web フロントエンドへの移行について issue を開いてご相談ください。

## 機能

上部の5つのタブ（+ 右側のチャットパネル）：指導教員を読む、最新論文を追跡、プロジェクト審査、数式検証、論文執筆。

### 指導教員を読む

指導教員の言葉を貼り付けます。実行可能なタスクに自動分解し、優先順位と所要時間の見積もりを含めます。各タスクは展開してサブステップと推奨リソースを表示できます。指導教員のデータは機密レベルに分類され、ローカルモデルの使用が強制されます。

### 最新論文を追跡

研究キーワードを設定します。ArXiv、Semantic Scholar、DBLP を並列にクロールします。論文は情報フィードとして表示され、ワンクリックで中国語の要約を取得できます。バックグラウンドで1時間ごとに自動更新されます。

三重の重複除去（タイトル類似度 + arxiv_id + DOI）により、プレプリントの重複表示を防ぎます。「発想を探す」をクリックすると、10以上の観点（理論性、効率性、頑健性、公平性など）から現在の論文リストを分析し、信頼度順に研究ギャップを表示します。

### プロジェクト審査

プロジェクト名と説明を提出します。3つの観点（革新性、厳密性、方法論）から評価し、具体的な弱点と改善提案を提示します。

### 数式検証

左右二段組で LaTeX ソースコードとレンダリングプレビューを表示します。提出後、2チャンネルで検証：基本チェック（括弧の対応、ゼロ除算、定義域）+ SymPy による数式処理。両方のチャンネルが合格して初めて緑信号が点灯します。LLM は一切使用しません — 完全ローカルの数式処理です。

### 論文執筆

構造化されたアウトラインを生成し、推定文字数を含めます。BibTeX の貼り付け解析をサポートします。下部パネルには「AI臭除去」ツールがあり、AIによる執筆痕跡を検出して自動的に除去します。

## プラグインシステム

プラグインは `~/.yanmo/plugins/` に配置します。設定 → プラグインマネージャーからロードします。最低要件：`plugin.toml` + `plugin.py`。開発ガイドは `plugin_schema/API.md` を参照してください。ライフサイクル管理、ホットロード/アンロード、イベントバスをサポートしています。

## アーキテクチャ

| コンポーネント | 説明 |
|---|---|
| プラグインエンジン | ライフサイクル管理、ホットロード/アンロード、イベントバス |
| タスクスケジューラー | 1時間ごとの自動論文取得 |
| WebSocket | 論文更新をフロントエンドにリアルタイムプッシュ |
| セキュリティ層 | 3段階のデータ分類 + クラウド操作のブロック + 監査ログ |
| ストレージ | SQLite（構造化データ）+ ChromaDB（ベクトル）+ ローカルファイル |

## データセキュリティ

デフォルト：すべての操作はローカルの Ollama を使用します。クラウド API キーはオプションです — 入力しなければ、あなたのマシンからデータが外部に出ることは一切ありません。

- 機密データはローカル強制、クラウド操作は自動ブロック
- クラウドへの送信トラフィックは監査ログに記録
- データは `~/.yanmo/` に保存され、いつでもバックアップ可能
